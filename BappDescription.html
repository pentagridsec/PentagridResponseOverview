Response Overview groups HTTP response bodies by similarity and presents one request/response per group. This provides security testers with an efficient method to identify anomalies, exotic error messages, and overlooked functionality across all responses captured from any Burp tool.

Features

  • Automatically groups similar responses using a similarity threshold to reduce noise and highlight unique responses
  • Identifies vulnerabilities missed by automated scanners by surfacing unusual error messages and unexpected responses
  • Removes reflected request parameters before comparison to prevent false grouping from parameter reflection
  • Processes responses from all Burp tools in real-time with configurable filtering by scope, file extension, and MIME type
  • Provides sortable tabular view with status codes, URLs, response sizes, and group sizes for quick analysis

Usage

  1. Add your target website to Burp Suite's scope
  2. Test the web application using any Burp tools (Proxy, Scanner, Intruder, Repeater)
  3. Navigate to the Response Overview tab to review grouped responses, sorted by any column
  4. Examine responses with small group sizes or unusual status codes for potential security issues
  5. Right-click and select "Hide item(s)" to remove reviewed entries from the overview
